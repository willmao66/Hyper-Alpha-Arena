# Binance Account Status Display on Dashboard

## Goal
在 Dashboard 底部的账户信息区域显示 Binance 账户信息（与 Hyperliquid 并列），包括 Detail 弹窗中的 API 权重信息。

## Changes

### 1. Backend: BinanceTradingClient - 保存 rate limit 信息
**File**: `backend/services/binance_trading_client.py`
- `_request()` 方法中保存 `x-mbx-used-weight-1m` 到实例变量 `self._last_used_weight`
- 新增 `get_rate_limit()` 方法，返回 `{ used_weight, weight_cap: 2400, remaining, usage_percent }`

### 2. Backend: binance_routes.py - 新增 summary API
**File**: `backend/api/binance_routes.py`
- 新增 `GET /api/binance/accounts/{account_id}/summary`
- 返回: equity, margin_usage, api_weight (从 get_rate_limit), positions count/list
- 新增 `GET /api/binance/accounts/{account_id}/rate-limit`
- 返回: used_weight, weight_cap, remaining, usage_percent

### 3. Backend: arena_routes.py - 扩展 positions 端点
**File**: `backend/api/arena_routes.py`
- 修改 `get_positions_snapshot()`: testnet/mainnet 模式下，除了查 HyperliquidWallet 的账户外，也查 BinanceWallet 的账户
- 为 Binance 账户调用 `BinanceTradingClient.get_balance()` 和 `get_positions()` 获取实时数据
- 返回的 account 数据中增加 `exchange: "hyperliquid" | "binance"` 字段

### 4. Frontend: HyperliquidMultiAccountSummary - 支持多交易所
**File**: `frontend/app/components/portfolio/HyperliquidMultiAccountSummary.tsx`
- accounts prop 增加 `exchange` 字段
- 根据 exchange 字段区分数据获取逻辑:
  - hyperliquid: 保持现有逻辑 (getHyperliquidBalance, getWalletRateLimit)
  - binance: 调用 `/api/binance/accounts/{id}/summary`
- 卡片标题旁显示交易所 SVG logo
- API 配额区域: hyperliquid 显示 "XX% left", binance 显示 "Weight: XX/2400"
- Win Rate: binance 暂时显示 "N/A"

### 5. Frontend: TraderDetailModal - 支持 Binance
**File**: `frontend/app/components/portfolio/TraderDetailModal.tsx`
- Account Status: binance 不显示 Wallet Address，改为显示 API Key (masked)
- API Usage: binance 显示每分钟权重 (used/2400)，标注 "Resets every minute"
- Trading Stats: binance 暂时显示 "Coming Soon"
- Positions: 复用现有持仓显示逻辑

### 6. Frontend: i18n 翻译
**Files**: `frontend/app/locales/en.json`, `frontend/app/locales/zh.json`
- 新增 Binance 相关翻译键
